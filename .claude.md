# Appointment Booking API — Project Reference

## Overview
Production-grade appointment/booking system for small businesses (clinic, salon, gym, consultant). Portfolio reference project.

## Tech Stack
- Python 3.12, Django 5.0.x, DRF 3.15.x
- PostgreSQL 16, JWT auth (simplejwt)
- drf-spectacular (OpenAPI/Swagger/ReDoc)
- django-filter, django-cors-headers, django-environ
- Docker + docker-compose (web + postgres)
- pytest + pytest-django + factory_boy
- black + ruff for formatting/linting

## Project Structure
```
config/           — settings (base/dev/prod), urls.py, wsgi/asgi
apps/
  accounts/       — Custom User (role: customer/staff/admin), auth endpoints, admin
  services/       — Service model, CRUD, admin
  staff/          — StaffProfile, WorkingHours, TimeOff, availability endpoint, admin
  appointments/   — Appointment model, booking logic, validators, admin
core/             — Shared utils, permissions, pagination, exceptions, seed command
tests/            — All tests (51), factories, conftest
requirements/     — base.txt, dev.txt, prod.txt
```

## Data Models
1. **User** — extends AbstractUser, role (customer/staff/admin), phone_number
2. **Service** — name, description, duration_minutes, price, is_active
3. **StaffProfile** — OneToOne(User), services_offered (M2M), bio
4. **WorkingHours** — staff(FK), weekday(0-6), start_time, end_time
5. **TimeOff** — staff(FK), start_datetime, end_datetime, reason
6. **Appointment** — customer(FK), staff(FK), service(FK), start/end_datetime, status, notes

## Core Business Logic
- Double-booking prevention with overlap queries (Q objects)
- Validates: no overlap, within working hours, not during time-off, correct duration
- Serializer-level validation: staff FK must have role='staff', customer FK must have role='customer'
- Error responses: 409 Conflict (overlap), 400 Bad Request (invalid time/outside hours)

## API Endpoints
- Auth: register, login (JWT pair), refresh, logout (blacklist), me
- Services: public list, admin CRUD
- Staff: list with services, availability by date
- Appointments: role-scoped list, create, detail, update, cancel, confirm
- All endpoints documented with @extend_schema (drf-spectacular)

## Roles & Permissions
- Customer: browse services, create/view/cancel own bookings
- Staff: view/manage own assigned bookings
- Admin: full access to everything
- Custom DRF permission classes: IsCustomerRole, IsStaffRole, IsAdminRole, IsOwnerOrStaffOrAdmin

## Django Admin
- Custom site header/title ("Booking System Admin")
- Colored role/status badges
- Admin actions: confirm, cancel, complete appointments
- Date hierarchy, search, filters on all models

## Limitations
- No real payment integration (price is display-only)
- No email/SMS notifications (Celery reminder is optional stretch)
- No frontend — API only
- No multi-tenant/organization support
- No real-time availability updates (no WebSockets)
- No i18n/l10n beyond UTC timezone
- Time slots computed per-request, not cached
